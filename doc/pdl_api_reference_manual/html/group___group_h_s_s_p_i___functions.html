en_result_t HsSpi_Init ( volatile FM_HSSPI_TypeDef *  pstcHsSpi,
const stc_hsspi_config_t *  pstcConfig 
)

Initialisation of the HS-SPI module.

FIFO width is limited to 8 bit and DMA is not supported in direct mode. The (high speed) SPI module is initialized in direct mode after call of Spi_Init().

Parameters
[in]pstcHsSpiHS-SPI module instance (register start address of HS-SPI module).
[in]pstcConfigHS-SPI configuration parameters.
Return values
OkHS-SPI module successfully initialised.
ErrorInvalidParameterIf one of the following conditions are met:
  • pstcHsSpi == NULL
  • pstcConfig == NULL
  • pstcConfig->u8TxFifoThresholdLow > (HS_SPI_MAX_FIFO_DEPTH - 1)
  • pstcHsSpiInternData == NULL (invalid or disabled HS-SPI unit (PDL_PERIPHERAL_ENABLE_HSSPIn))
ErrorIf HsSpi disable/enable failed

en_hsspi_protocol_direction_t  enProtocolDirection

Set mode of the HS-SPI unit.

Selects if HS-SPI unit shall be operated in Direct or Command Sequencer mode and which protocol mode (legacy, dual, quad) shall be used. Any transmission has to be completed before a change of HS-SPI mode is possible, otherwise any ongoing transmission will be interrupted.

Parameters
[in]pstcHsSpiHS-SPI module instance (register start address of HS-SPI module).
[in]enModeDirect mode or command sequencer mode.
[in]enProtocolModeHS-SPI Mode (Legacy, Dual or Quad)
[in]enProtocolDirectionHS-SPI protocol direction
Return values
OkNew mode was successfully set.
ErrorInvalidParameterIf following condition is met:
  • pstcHsSpi == NULL
  • pstcHsSpiInternData == NULL (invalid or disabled SPI unit (PDL_PERIPHERAL_ENABLE_HSSPIn))
ErrorIf HsSpi disable/enable failed

Configures the communication parameters of an external device.

The external device specific parameters (e.g. HS-SPI mode, LSB/MSB first, ...) can be configured In case of high speed SPI the slave u8DeviceNumber is also enabled for the memory map in command sequencer mode. (See HW-manual for more information)

  • pstcConfig == NULL
  • pstcHsSpiInternData == NULL (invalid or disabled HS-SPI unit (PDL_PERIPHERAL_ENABLE_HSSPIn))
  • u8DeviceNumber > 3

Transfers data in direct half-duplex mode.

Note that direct mode transfer protocol is changed during operation and previous mode is restored after operation has finished.

Note
This function does not work for HS-SPI clock modes 4-7
This function is restricted to HS-SPI protocol mode 'Legacy'. i.e. does not support half duplex transfer in case of 'Dual' or 'Quad' mode.
Parameters
[in]pstcHsSpiHS-SPI module instance (register start address of (high speed) SPI module).
[in]u8DeviceNumberSlave which is connected to corresponding signal SSEL0..3.
[in]pu8TxDataPointer to transmit buffer
[in]u16TxSizeNumber of data bytes to transmit. Must be greater than 0.
[in]pu8RxDataPointer to receive buffer, can be NULL.
[in]u16RxSizeNumber of data bytes to receive, can be 0
[in]bEnableTxDmaRequestEnable or disable DMA request on TX FIFO interrupts (less or equal threshold level). -> currently not supported
Return values
OkSetup or transmission of data successful.
ErrorOperationInProgressA transmission is still ongoing.
ErrorInvalidModeHS-SPI module is in command sequencer mode. or protocol mode is not "SpiProtocolModeLegacy" or clock mode >= "SpiClkLowOutFallingInFalling"
ErrorInvalidParameterIf one of following conditions are met:
  • pstcHsSpi == NULL
  • pu8TxData == NULL
  • u8DeviceNumber > 3
  • u16TxSize == 0
  • u16TxSize + u16RxSize > (2^16 - 1)
  • pstcHsSpiInternData == NULL (invalid or disabled HS-SPI unit (PDL_PERIPHERAL_ENABLE_HSSPIn))

en_result_t HsSpi_MultiModeTransfer ( volatile FM_HSSPI_TypeDef *  pstcHsSpi,
uint8_t  u8DeviceNumber,

uint8_t  u8LegacyTxSize,
const uint8_t *  pu8TxData,

)

Starts transfer in legacy mode and then switches to quad mode.

Some HS-SPI devices may require an access sequence where the first part of a command is transmitted in legacy mode and afterwards further parameters or the response from the device are transmitted in quad mode. This function allows to execute such an SPI transaction.

Note that direct mode transfer protocol is changed during operation and previous mode is restored after operation has finished.

Parameters
[in]pstcHsSpiHS-SPI module instance (register start address of high speed SPI module).
[in]u8DeviceNumberSlave which is connected to corresponding SSEL0..3 line
[in]pu8LegacyDataPointer to transmit buffer for legacy mode
[in]u8LegacyTxSizeNumber of data bytes to transmit in legacy mode
[in]pu8TxDataPointer to transmit buffer for quad mode, can be NULL.
[in]u8TxSizeNumber of data bytes to transmit in quad mode Valid range from 0 to HS_SPI_MAX_FIFO_DEPTH.
[in]pu8RxDataPointer to receive buffer for quad mode, can be NULL.
[in]u8RxSizeNumber of data bytes to receive in quad mode. Valid range from 0 to HS_SPI_MAX_FIFO_DEPTH.
Return values
OkSetup or transmission of data successful.
ErrorOperationInProgressu8TxSize > 0 and a transmission is still ongoing.
ErrorInvalidModeSPI module is in command sequencer mode.
ErrorInvalidParameterIf one of following conditions are met:
  • pstcHsSpi == NULL
  • pu8TxData == NULL
  • u8DeviceNumber > 3
  • u8TxSize == 0
  • u8RxSize > HS_SPI_MAX_FIFO_DEPTH
  • u8RxSize > 0 && pu8RxData == NULL
  • u8LegacyTxSize > 0 && pu8LegacyData == NULL
  • pstcHsSpiInternData == NULL (invalid or disabled HS-SPI unit (PDL_PERIPHERAL_ENABLE_HSSPIn))
en_result_t HsSpi_SetReadCommandSequence ( volatile FM_HSSPI_TypeDef *  pstcHsSpi,
const stc_hsspi_command_sequencer_control_t *  pstcCmdSequence 
)

Set read command sequence for command sequencer mode.

Every command sequence is either 8 commands long, or has to be finished with command HsSpiCmdEndOfList. To find out if a 8 commands long sequence is valid or not, the 9th command (which will not be transferred) must be a HsSpiCmdEndOfList command. The new sequence will be set after internally disabling the HS-SPI module. After update, the SPI module is enabled again.

Parameters
[in]pstcHsSpiHS-SPI module instance (register start address of high speed SPI module).
[in]pstcCmdSequencePointer to command sequence list (maximum 9 entries, last entry must be SpiCmdEndOfList).
Return values
OkRead command sequence successfully set.
ErrorInvalidModeHS-SPI module is in direct mode.
ErrorInvalidParameterIf one of following conditions are met:
  • pstcCmdSequence == NULL
  • SpiCmdEndOfList command missing in pstcCmdSequence
  • pstcHsSpi == NULL
ErrorIf HsSpi disable/enable failed
